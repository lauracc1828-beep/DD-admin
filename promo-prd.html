一、模块概述

推广渠道管理模块用于管理平台所有对外推广渠道的配置，包含渠道创建与编辑、推广商管理、推广效果查看、安装引导配置等功能。商户可在此模块为每条推广渠道生成唯一的带参落地链接，配置落地页类型、安装引导策略和第三方广告平台埋点回传，并通过推广效果报表实时查看各渠道的转化数据。

二、推广链路说明

以下流程图展示了从商户配置渠道到用户转化、数据回传的完整链路。

① 商户在后台配置推广渠道（落地指向 / 跳转域名 / 埋点类型 / 安装引导）
↓
② 系统生成唯一渠道链接（格式：跳转域名 + ?ch=渠道ID）
↓
③ 运营将渠道链接投放至推广渠道（短信 / Facebook / TikTok / 快手等）
↓
④ 用户点击推广链接，渠道ID（ch参数）随请求传递
↓
⑤ 跳转域名接收请求，根据落地指向将用户引导至对应落地页
↓ 落地页分两类
落地指向：仿谷歌页 / 防封下载页
⑥A 下载页（仿谷歌/苹果商店样式）
用户只有下载操作，无H5跳转

⑥B H5页面

↓
⑦ 用户完成安装（PWA / APK）/ 注册 / 充值等行为
↓ 数据出口分两路
所有渠道
⑧A 我方后台统计
按渠道ID（ch）归因
→ 推广效果报表展示
（注册/充值/提现/投注等）
配置了三方埋点的渠道

向Facebook/TikTok/Kwai
推送转化事件
（用于广告平台优化投放）

关键概念说明

概念说明
渠道ID（ch参数）系统为每条渠道分配的唯一数字标识，附加在渠道链接URL参数中，用于所有后续用户行为的数据归因。APK包本身不内嵌渠道ID，渠道信息通过下载URL参数在安装时透传
跳转域名推广链接的入口域名，负责接收用户点击请求并根据落地指向转发。来源于系统管理-域名管理中已配置并生效的推广域名
落地指向决定用户最终看到的页面类型，共5个选项，本质上分为「下载页类」和「H5页类」两大类，详见子页面一字段说明
埋点回传当前支持三种三方API回传：Facebook API / TK API / Kwai API，均为服务器端回传，不依赖浏览器，数据准确性高。回传参数（Pixel ID / Token）由投流方提供
直推 vs 裂变直推：用户自己点击推广链接进入；裂变：已注册用户邀请带来的新用户。两者数据在推广效果报表中分开统计
充提差充值金额 - 提现金额，是衡量渠道质量的核心指标。正值表示渠道用户为平台贡献利润，负值表示亏损

埋点类型说明

当前平台支持4种埋点类型：常规推广（无三方回传）+ Facebook API / TK API / Kwai API（三方服务器端API回传）。三方埋点的Pixel ID和Token均由投流方提供。
埋点类型展开字段说明
常规推广无不接入任何第三方广告平台，数据仅在我方后台统计
Facebook API（Facebook服务器接口回传）Pixel ID 必填、Token 必填服务器端回传至Facebook广告平台；具体回传事件以Facebook官方文档为准
TK API（TikTok服务器接口回传）Pixel ID 必填、Token 必填服务器端回传至TikTok广告平台；具体回传事件以TikTok官方文档为准
Kwai API（快手服务器接口回传）Pixel ID 必填、Token 必填服务器端回传至快手广告平台；具体回传事件以快手官方文档为准

事件说明弹窗格式：标题为「[埋点类型名称]」，内容为该类型的回传事件列表，每条格式为「中文事件名称 - 事件标识符」，具体内容以三方平台官方文档为准。

↳ 弹框间隔时间条件必填下拉单选引导安装开启后展示；可选：0小时(强制安装) / 1~24小时「0小时(强制安装)」：弹窗强制弹出，用户无法关闭，必须完成安装或继续停留在弹窗上。1~24小时：弹窗弹出后用户可关闭，但每隔对应时长会再次弹出，直到用户完成安装。
↳ 弹框安装类型条件必填下拉单选引导安装开启后展示；当前平台APK暂未上线，含APK选项实际置灰不可用「pwa(快速安装)」：引导用户将H5页面添加到手机桌面（PWA），若用户使用非谷歌浏览器，会提示用户用谷歌浏览器打开后再安装。「apk(需要自己封装)」：弹窗引导用户下载并安装APK包，需要商户提前上传封装好的APK。「pwa(优先)+apk(推荐使用)」：优先引导安装PWA，若当前浏览器不支持PWA，则降级引导安装APK。
↳↳ APK包地址条件必填文本输入弹框安装类型含APK时展示APK文件的下载地址，用户点击安装时从该地址下载APK包。
↳↳ APK包名—自动填充含APK时展示；选择马甲包后自动读取，手动修改APK包地址后自动清空APK的唯一包标识符，系统用于判断用户设备是否已安装该APK。
↳↳ 选择APK条件必填按钮组「选择马甲包」从系统管理-马甲包管理中选（自动填入地址和包名）；「上传」直接上传apk文件（限100MB）马甲包是经过伪装的APK，外观与普通APP相似，用于规避应用商店审核。选择马甲包后APK包地址和包名自动填入。
安装pwa/App赠送必填下拉单选可选：关闭 / 充值用户赠送 / 无需充值赠送「关闭」：用户安装后不赠送任何彩金。「充值用户赠送」：仅有过充值记录的用户完成安装后，系统自动赠送彩金到账户，未充值用户不赠送。「无需充值赠送」：所有完成安装的用户均可获得赠送彩金，包括未充值用户。赠送彩金到账后需满足打码倍数要求方可提现。
↳ 赠送金额条件必填数字输入选赠送类型后展示用户安装后实际到账的彩金金额，与弹窗中「显示赠送金额」相互独立，两者可以不同。
↳ 打码倍数条件必填数字输入选赠送类型后展示赠送彩金的流水要求倍数。例如赠送100元、打码倍数为10，则用户需累计投注1000元后，赠送金额方可提现。
用户完成PWA或APK安装后，H5首页顶部下载提示横幅不再显示，强制安装弹窗也不再弹出。

用于在同一地区/商户下一次性创建多条渠道。各渠道共享埋点类型和渠道设置（字段与新增渠道弹窗完全一致），渠道名称、落地指向、跳转域名、推广商可逐行独立配置。

固定字段：地区 / 所属商户 / 埋点类型 / 渠道设置（安卓/苹果Tab，字段与新增渠道完全一致）

渠道信息行（可动态增减，默认2行）

字段说明
渠道名称每条渠道独立填写
落地指向每条渠道独立选择，选项与新增渠道一致
跳转域名随落地指向联动，每条渠道独立选择

3.5.5 编辑渠道弹窗

点击表格操作列编辑图标后弹出，标题为「编辑渠道」。地区和所属商户字段置灰不可编辑，回填当前数据；渠道名称等其他字段均回填当前配置数据，渠道名称输入框右侧显示清除按钮（×）。所有配置项与新增渠道弹窗完全一致，提交后保存更新。

字段说明

3.5.7 事件说明弹窗

点击表格操作列「事件说明」图标后弹出（仅非常规推广类型的渠道显示此按钮）。弹窗标题为该埋点类型名称，内容展示该类型的回传事件列表，每条格式为「中文事件名称 - 事件标识符」。具体事件内容以三方平台官方文档为准。
