一、模块概述
推广渠道管理模块用于管理平台所有对外推广渠道的配置，包含渠道创建与编辑、推广商管理、推广效果查看、安装引导配置等功能。商户可在此模块为每条推广渠道生成唯一的带参落地链接，配置落地页类型、安装引导策略和第三方广告平台埋点回传，并通过推广效果报表实时查看各渠道的转化数据。

二、推广链路说明
以下流程图展示了从商户配置渠道到用户转化、数据回传的完整链路。
① 商户在后台配置推广渠道（落地指向 / 跳转域名 / 埋点类型 / 安装引导）
↓
② 系统生成唯一渠道链接（格式：跳转域名 + ?ch=渠道ID）
↓
③ 运营将渠道链接投放至推广渠道（短信 / Facebook / TikTok / 快手等）
↓
④ 用户点击推广链接，渠道ID（ch参数）随请求传递
↓
⑤ 跳转域名接收请求，根据落地指向将用户引导至对应落地页
↓ 落地页分两类
落地指向：仿谷歌页 / 防封下载页
⑥A 下载页（仿谷歌/苹果商店样式）
用户只有下载操作，无H5跳转
下载按钮和引导安装弹窗不生效
弹框安装类型决定安装PWA还是APK
落地指向：官网首页 / 官网注册页 / 主域名
⑥B H5页面
下载按钮横幅生效
引导安装弹窗生效
按渠道配置触发时机和安装类型
↓
⑦ 用户完成安装（PWA / APK）/ 注册 / 充值等行为
↓ 数据出口分两路
所有渠道
⑧A 我方后台统计
按渠道ID（ch）归因
→ 推广效果报表展示
（注册/充值/提现/投注等）
配置了三方埋点的渠道
⑧B 服务器端API回传
向Facebook/TikTok/Kwai
推送转化事件
（用于广告平台优化投放）
关键概念说明
概念
说明
渠道ID（ch参数）
系统为每条渠道分配的唯一数字标识，附加在渠道链接URL参数中，用于所有后续用户行为的数据归因。APK包本身不内嵌渠道ID，渠道信息通过下载URL参数在安装时透传
跳转域名
推广链接的入口域名，负责接收用户点击请求并根据落地指向转发。来源于系统管理-域名管理中已配置并生效的推广域名
落地指向
决定用户最终看到的页面类型，共5个选项，本质上分为「下载页类」和「H5页类」两大类，详见子页面一字段说明
埋点回传
当前支持三种三方API回传：Facebook API / TK API / Kwai API，均为服务器端回传，不依赖浏览器，数据准确性高。回传参数（Pixel ID / Token）由投流方提供
直推 vs 裂变
直推：用户自己点击推广链接进入；裂变：已注册用户邀请带来的新用户。两者数据在推广效果报表中分开统计
充提差
充值金额 - 提现金额，是衡量渠道质量的核心指标。正值表示渠道用户为平台贡献利润，负值表示亏损

埋点类型说明
当前平台支持4种埋点类型：常规推广（无三方回传）+ Facebook API / TK API / Kwai API（三方服务器端API回传）。三方埋点的Pixel ID和Token均由投流方提供。
埋点类型
展开字段
说明
常规推广
无
不接入任何第三方广告平台，数据仅在我方后台统计
Facebook API（Facebook服务器接口回传）
Pixel ID 必填、Token 必填
服务器端回传至Facebook广告平台；具体回传事件以Facebook官方文档为准
TK API（TikTok服务器接口回传）
Pixel ID 必填、Token 必填
服务器端回传至TikTok广告平台；具体回传事件以TikTok官方文档为准
Kwai API（快手服务器接口回传）
Pixel ID 必填、Token 必填
服务器端回传至快手广告平台；具体回传事件以快手官方文档为准
事件说明弹窗格式：标题为「[埋点类型名称]」，内容为该类型的回传事件列表，每条格式为「中文事件名称 - 事件标识符」，具体内容以三方平台官方文档为准。
