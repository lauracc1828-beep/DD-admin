一、需求背景

会员答谢活动是平台针对会员按周期定期发放答谢奖励的留存激励活动。运营人员可配置每日、每周或每月的会员日，并为每个会员日设置基于充值、盈亏或有效投注等维度的阶梯奖励规则。会员在会员日当天手动前往活动详情页领取奖励。

本活动适用于以下场景：

  • 日常会员留存：通过固定周期的奖励答谢，培养会员定期访问和参与平台的习惯，降低流失率。
  • 充值与活跃激励：以充值金额、盈亏或有效投注为奖励条件，引导会员在会员日前保持活跃，提升平台整体投注流水。
  • 分层精细化运营：支持按参与层级配置活动可见性，可针对不同价值段的会员推送差异化答谢力度。

二、名词解释

术语说明
重置方式活动的周期类型，决定会员日的节奏：每日（每天固定时间点）、每周（每周指定星期）、每月（每月指定日期）
会员日在活动周期内，系统开放奖励领取的具体时间节点。每日模式以时间点为单位；每周/每月模式以日期为单位，每个日期对应一张独立的奖励配置卡
奖励条件领取奖励前须满足的统计维度，包括：充值金额、盈亏、有效投注、无限制（无条件发放）
选择时间奖励条件的统计时间范围，如昨日、上周、本月等，决定系统用哪段时间的数据来判断是否达标
奖励类型奖励的计算方式：固定金额、随机金额（区间内随机）、固定比例（按条件金额的固定百分比）、随机比例（按条件金额的随机百分比区间）
稽核倍数领取奖励后，提款前须完成的打码量倍数。计算公式：奖励金额 × 稽核倍数 = 所需有效投注额
申领终端允许发起领取操作的客户端类型，仅手动领取时生效
申领限制除终端外的额外申领门槛，如首充要求、IP唯一性限制等
参与层级按会员层级筛选活动可见范围，不在层级内的会员将看不到该活动

四、后台功能说明

4.1 活动列表页

路径：后台管理 > 福利中心

本需求涉及以下三个操作入口：

  • 新增活动：点击"新增活动"按钮，弹出活动类型选择弹框，选择"会员答谢"后进入配置弹框（详见 4.2）。
  • 查看活动：点击列表中的"查看"操作，以只读方式展示活动完整配置，字段与新增/编辑弹框一致，不可修改（详见 4.4）。
  • 修改活动：点击列表中的"编辑"操作，进入与新增相同的配置弹框，字段可修改。已生效活动的活动分类与活动类型不可修改。

4.2 新增/编辑活动弹框

弹框为单一滚动表单，字段按以下顺序展示。字段名后标注 通用 表示所有活动类型共用的配置项；标注 专属 表示会员答谢活动特有的配置项。

字段类型说明
活动分类 通用 下拉选择 读取系统活动分类配置，下拉选择。编辑已生效活动时不可修改。
活动类型 通用 下拉选择 固定选择"会员答谢"，决定活动的业务逻辑和配置项。编辑已生效活动时不可修改。
活动名称 通用 单选 + 文本 系统默认：自动填入"会员答谢活动"，前台展示该名称；自定义：可手动输入活动名称，前台将展示自定义名称
活动时间 通用 单选 + 日期 永久：活动长期有效，无结束时间；自定义：需填写活动时间范围（控制活动是否有效）及展示时间范围（控制活动在前台的显示时段，可与活动时间不同）
重置方式 专属 单选 决定会员日的时间维度和循环节奏：
• 每日：每天固定在指定时间点开放奖励领取，选择后需配置"时间"字段（0～23点，整点选择）
• 每周：每周的指定星期开放奖励领取，选择后需通过"添加会员日"配置具体星期（周一～周日，可多选）
• 每月：每月的指定日期开放奖励领取，选择后需通过"添加会员日"配置具体日期（1～31日，可多选）
时间 专属 下拉选择 仅重置方式为"每日"时显示。配置每日会员日的开放时间，可选 0～23 点（整点），代表该整点时刻起会员可在前台领取当日奖励。
添加会员日 专属 多选标签 仅重置方式为"每周"或"每月"时显示。
• 每周模式：从周一至周日中多选，选中的星期即为会员日；可配置多个星期，每个星期对应一套独立的奖励配置
• 每月模式：从1日至31日中多选，选中的日期即为会员日；可配置多个日期，每个日期对应一套独立的奖励配置
派发方式 通用 单选 自动派发：系统在会员满足奖励条件时自动将奖励发放至账户（会员答谢固定使用此方式）；玩家申请：会员需主动发起申请后才能领取（当前不支持，选项置灰）
审核方式 通用 单选 系统(周期内)：在活动周期内由系统自动完成审核并派发奖励（会员答谢固定使用此方式）；系统(过期时)：到达奖励过期时间时系统统一审核（当前不支持）；人工审核：由运营人员手动审核后派发（当前不支持）
派奖时间 通用 单选 立即到账：奖励审核通过后即时发放至会员账户（会员答谢固定使用此方式）；延迟到账：在指定时间后到账（当前不支持）
领取方式 通用 单选 手动领取：会员需在活动详情页主动点击"领取"按钮，系统再进行审核派发（会员答谢固定使用此方式，选中后下方显示申领终端与申领限制配置）；自动领取：系统满足条件后自动发放，无需会员操作（当前对会员答谢不支持，选项置灰）
奖励作废 通用 单选 + 数字 控制会员日过期后奖励的处理方式：
• 过期自动派发：到达过期时间后，不满足条件的奖励自动派发给会员（会员答谢可选）
• 过期作废：到达过期时间后，不满足条件的奖励自动作废（会员答谢可选）
• 奖励保留天数自动派发：在设定的保留天数内满足条件后自动派发奖励，选中后需填写保留天数（会员答谢暂不支持，选项置灰）
• 奖励保留天数作废：超过保留天数后不满足条件则奖励自动作废，选中后需填写保留天数（会员答谢暂不支持，选项置灰）
稽核倍数 通用 数字输入 奖励金额 × 稽核倍数 = 提款所需打码量，全局统一配置，适用于该活动下所有会员日
参与条件 通用 单选 全部会员：所有已登录会员均可参与；新注册会员(活动时间内注册玩家)：仅在活动有效时间内完成注册的会员可参与，选中后系统自动备注"新会员以注册时间自动报名参加活动"
显示最高可获得奖励 专属 数字输入 在前台活动详情页展示的最高奖励金额，用于吸引会员参与。该值为展示用途，非实际可领取金额上限，由运营根据奖励配置合理填写。
奖励配置 专属 复合配置 为每个会员日配置奖励规则，详见 4.3 章节。
申领终端 通用 多选 仅领取方式为"手动领取"时显示。限制允许发起领取操作的终端类型，可选：苹果H5 / 安卓H5 / PWA。未勾选时不限制终端类型。
申领限制 通用 多选 + 条件输入 仅领取方式为"手动领取"时显示。各选项说明：
• 绑定收款方式：账号需绑定收款方式才能领取奖励（系统暂不支持，始终置灰不可选）
• 完成首充可申领：账号需完成首次充值，才能领取奖励
• 同姓名只能申领1次：若不同账号使用相同真实姓名，某个账号领取奖励后，其他账号在同一活动中无法领取（系统暂不支持，始终置灰不可选）
• 同登录IP只能申领1次：若不同账号登录时的IP相同，某个账号领取奖励后，其他账号在同一活动中无法领取
• 同注册IP只能申领1次：若不同账号注册时的IP相同，某个账号领取奖励后，其他账号在同一活动中无法领取
• 最近N天内充值次数≥N次：领奖当日前N天（含当天）内累计充值N次，才能领取奖励
• 最近N天内充值金额≥N：领奖当日前N天（含当天）累计充值金额满N元，才能领取奖励
注：「绑定收款方式」与「同姓名只能申领1次」因系统暂不支持，始终置灰不可选。
稽核限定平台 通用 单选 + 多选 不限制：稽核打码量计算所有游戏平台的有效投注；仅限以下平台：稽核打码量仅计算选定游戏平台的有效投注，可按电子/体育分类选择具体平台
参与层级 通用 多选 控制活动的可见范围。可选择全部层级，或从系统层级列表中勾选指定层级（如：默认层级、10雷、20雷等）。同时支持排除特定层级（如：不出款、测试账号、刷子套利等），被排除的层级即便勾选也无法参与。未在可见层级中的会员将看不到该活动入口。
规则说明 通用 单选 + 文本域 系统默认：使用平台内置的活动规则文案，自动展示在前台活动详情页；自定义：运营人员手动编辑规则内容，替换系统默认文案
显示排序 通用 数字输入 控制活动在福利中心列表中的展示顺序，数字越小排序越靠前
活动详情页选择 通用 单选 跟随皮肤：活动详情页样式跟随商户整体UI皮肤设置；样式一：使用固定的活动详情页模板，选择后可点击"预览样式"查看效果
背景样式 通用 单选 + 选图/上传 系统默认：从预设渐变色图库中选择背景图；自定义：上传自定义背景图片（建议尺寸 660×220px，支持 JPG/PNG/SVG）
LOGO样式 通用 单选 + 选图/上传 仅背景样式为"系统默认"时显示。系统默认：按活动分类从图标库中选择图标；自定义：上传自定义图标（建议尺寸 300×300px）
宣传图预览 通用 实时预览 实时合成展示背景 + LOGO + 活动名称的叠加效果，用于确认视觉呈现是否符合预期

4.3 奖励配置（专属）

奖励配置区域跟随重置方式的不同呈现不同结构：

每日模式

整个活动只有一套奖励配置，不分会员日卡片，直接展示以下字段：

子字段说明
奖励配置（条件维度） 选择用于判断是否达到领取门槛的统计维度：
• 充值：以选定时间段内的充值金额作为阶梯条件
• 盈亏：以选定时间段内的盈亏金额（输赢差值）作为阶梯条件
• 有效投注：以选定时间段内的有效投注金额作为阶梯条件
• 无限制：无条件直接发放，不需要统计任何数据；选择此项后"选择时间"字段隐藏
选择时间 奖励条件的统计时间范围，仅当条件维度不为"无限制"时显示。可选项：历史-会员日（活动开启至今的历史数据）、当日-会员日（当天数据）、昨日、上周、上月、近七天(不含当日)、近三十天(不含当日)、自定义时间（指定近N天）
奖励类型 奖励金额的计算方式：
• 固定金额：满足对应阶梯条件时发放固定金额
• 随机金额：满足对应阶梯条件时在设定的最小值～最大值区间内随机发放
• 固定比例：按条件金额（充值/盈亏/投注）的固定百分比发放，如充值的5%
• 随机比例：按条件金额的随机百分比区间发放，如充值的3%～8%
条件→奖励表 阶梯配置表，每行为一个档位：左列填写条件金额阶梯（如充值金额 ≥ 100），右列填写对应奖励金额或比例（根据奖励类型不同展示单列或双列）。可新增/删除行。

每周 / 每月模式

系统为每个已添加的会员日生成一张独立的奖励配置卡片（如：会员日 1、会员日 2……）。每张卡片内包含与每日模式相同的四个子字段（奖励配置、选择时间、奖励类型、条件→奖励表），各会员日之间的奖励配置相互独立，不互相影响。

📌 会员日卡片的数量与顺序由"添加会员日"字段中选中的日期/星期决定，选中几个就生成几张卡片，系统按时间顺序自动排列。

4.4 查看活动弹框

点击列表中"查看"进入，以只读形式展示活动的完整配置内容。所有字段与新增/编辑弹框一致，但均不可编辑。用于运营人员快速核查活动配置详情，无需进入编辑模式。

五、前台功能说明

5.1 入口与页面结构

前台活动详情页由活动 Banner（宣传图）、活动说明区（显示最高可获得奖励、活动规则）、领取按钮区三部分组成。会员进入福利中心后点击会员答谢活动卡片进入详情页。

页面顶部展示活动倒计时或当前会员日状态；底部固定显示"领取"按钮，按钮状态随活动逻辑实时变化。

5.2 状态说明

前台按钮有两种视觉状态：激活（可点击）和置灰（不可点击），具体样式以UI设计为准。以下为完整的前台状态列表：

状态触发条件页面表现
未登录 会员未登录时访问活动页 领取按钮置灰，点击引导会员登录
已登录 · 非会员日（倒计时中） 当前时间不在任何会员日开放时段内 页面显示距下次会员日的倒计时，领取按钮置灰不可点
已登录 · 会员日 · 可领取 当前为会员日，且满足奖励条件，且未领取 领取按钮激活，会员可点击领取
已登录 · 会员日 · 已领取 当前为会员日，会员已完成领取 领取按钮置灰不可点，与"不满足条件"视觉样式相同
已登录 · 会员日 · 不满足条件 当前为会员日，但会员未满足奖励配置中的阶梯条件（充值/盈亏/投注不达标） 领取按钮置灰不可点，与"已领取"视觉样式相同
ℹ️ "已领取"与"不满足条件"两个状态的按钮视觉样式完全一致（均为置灰），系统不在按钮上区分两者；实际发放状态可在会员的奖励记录页查看。

5.3 领取交互

会员点击激活状态的"领取"按钮后，系统验证申领条件（终端限制、申领限制等）：

  • 条件不满足：页面展示悬浮提示卡片，说明未满足的具体申领条件（如：请通过 iOS H5、Android H5、PWA 申领）。
  • 条件满足：弹出领取成功弹窗，展示本次获得的奖励金额（如：恭喜您获得 R$X.XX 奖金），会员点击"确认"关闭弹窗，按钮变为已领取态（置灰）。
