一、模块概述

安装引导模块用于配置官网主域名（不带渠道 ID）下的下载按钮和安装引导弹框行为，面向未通过推广渠道链接访问站点的用户。

本模块配置与各推广渠道的渠道设置相互独立、互不影响，各自生效。推广渠道的引导安装配置详见《推广渠道管理模块 PRD》。

项目说明
菜单位置商户后台 → 推广渠道管理 → 安装引导
生效范围下载按钮与引导安装仅对落地指向为 H5 页类（官网首页 / 官网注册页）的访问生效；落地指向为下载页类（仿谷歌页 / 仿苹果商店页）时，下载按钮与引导安装不生效，但安装类型、安装包地址等配置字段仍生效。
配置粒度以商户为单位配置

二、后台配置

2.1 页面行为

  • 页面进入后直接展示当前商户的默认配置，无需点击查询
  • 切换地区或商户后点击「查询」，刷新并展示所选商户的当前配置
  • 修改配置后点击「保存」提交，保存成功后给出轻提示

2.2 筛选区

地区（下拉单选，默认当前地区）/ 商户（下拉单选，随地区联动）/ 查询按钮

2.3 配置字段

字段值持久化说明：「引导安装」开关仅控制下方子字段的展开与收起，不重置或清空字段内容。无论开关处于何种状态，字段始终保留其最后一次编辑的值（或从未编辑时的系统默认值）。

三、前台显示逻辑

3.1 生效条件

落地指向类型下载按钮引导安装安装类型 / 安装包地址等
下载页类（仿谷歌页 / 仿苹果商店页）❌ 不生效❌ 不生效✅ 生效
H5 页类（官网首页 / 官网注册页）✅ 生效✅ 生效✅ 生效
下载页类（仿谷歌页 / 仿苹果商店页）的页面区块、样式与下载交互统一以《仿应用商店模块 PRD》为准。

3.2 下载按钮（顶部横幅）

下载按钮开启时，H5 首页顶部展示下载提示横幅（流布局，展开时将页面内容向下推移，不覆盖页面）。

横幅内容：APP 图标（来源：系统管理 → 系统配置 → 站点配置 → APP 图标）+ 固定文案（见第五章中葡对照）+ 装饰图（静态资源）+ 安装按钮 + 关闭按钮（「×」，点击后当次会话不再显示）。

点击安装按钮，进入对应设备的安装流程（见 3.4 / 3.5 节）。

底部安装浮层出现时，横幅被遮罩层覆盖，不被关闭。用户点击安装按钮并完成安装流程后，顶部横幅保持可见，不自动隐藏。

3.3 底部安装浮层

下载按钮开启时，在 H5 页面按弹框时间点触发底部安装浮层。浮层出现时带遮罩层，覆盖在整个页面上方（包括顶部横幅）。

触发优先级：底部安装浮层优先级低于首页其他弹窗（活动弹窗、公告弹窗等）。首页存在其他弹窗时，底部安装浮层不显示，仅展示顶部横幅。

内容说明：APP 图标（来源：系统管理 → 系统配置 → 站点配置 → APP 图标）+ 说明文案（固定文案，含动态域名，见第五章中葡对照）+ 取消按钮 + 安装按钮。

域名展示规则：读取当前最终 URL 的 hostname 部分（去除 https:// 前缀及路径，保留纯域名字符串），如「v5kk678w.com」。

关闭逻辑：

  • 仅点击「取消」按钮可关闭浮层
  • 点击浮层外遮罩区域不关闭浮层
  • 浮层打开后 5 秒无操作自动关闭
关闭浮层后，遮罩消失，顶部横幅恢复可见。点击安装按钮后进入对应设备的安装流程（见 3.4 / 3.5 节）。

3.4 引导安装流程（安卓分支）

引导安装弹窗对安卓与 iOS 设备均生效，与下载按钮（顶部横幅 / 底部安装浮层）共用同一套安装交互逻辑（含设备 / 浏览器分支），仅入口形式不同。触发后从屏幕底部上推显示弹窗，带遮罩覆盖页面。本节描述安卓设备分支；iOS 设备分支与下载按钮一致，详见 3.5 节（Safari→整页进入仿苹果页并按其流程安装；非 Safari→弹出复制链接弹框）。iOS 设备下弹窗整块隐藏本节的「安卓应用安装说明」（步骤 + 友情提示）。

弹窗内容：顶部品牌插图（静态资源）+ 标题 + 副标题 + APP 信息行（APP 图标及站点名称，来源：系统管理 → 系统配置 → 站点配置）+ 安装按钮 + 客服按钮 + 安装步骤区 + 安全提示（文案见第五章中葡对照）。

客服按钮与安装按钮并排同行，点击后跳转客服页面；常规升级和赠送彩金升级状态下均显示。

赠送彩金升级状态：标题不变，副标题替换为含赠送金额的文案，金额动态读取后台「显示赠送金额」配置。

配置类型副标题展示格式
固定金额Baixe o aplicativo para o seu desktop e ganhe um bônus extra de R$[金额].
随机金额Baixe o aplicativo para o seu desktop e ganhe um bônus extra de R$[最小值]～R$[最大值].

下载进度状态

触发 APK 下载时，安装按钮区域替换为进度条，从 0% 动画递增至 100%。进度达到 100% 后，进度条区域恢复为安装按钮，同时 APK 文件下载完成（浏览器层面触发文件保存）。

进度条仅在配置含 APK 下载时出现；纯 PWA 配置点击安装后不显示进度条，直接触发系统 PWA 安装提示。

点击安装按钮后的交互流程

① 纯 PWA 配置

条件交互
浏览器支持 PWA（支持 beforeinstallprompt）触发系统 PWA 安装提示（带遮罩，覆盖下层页面及弹框，系统提示位于最上层）
• 确认 → 自动安装 PWA 并打开，引导弹窗保留
• 取消 → 关闭系统提示，引导弹窗保留
浏览器不支持 PWA弹出「复制链接到 Chrome」引导弹框，引导安装弹窗保留

② 纯 APK 配置

纯 APK 配置下，用户点击安装后直接触发下载，不弹出任何确认弹框（APK 确认弹框仅在 PWA+APK 配置且用户取消 PWA 提示后才出现，见③）。
步骤说明
点击安装按钮直接触发 APK 下载，按钮区域替换为进度条（0→100%）
进度 100%进度条恢复为安装按钮，APK 文件下载至设备存储

③ PWA（优先）+ APK 配置

APK 确认弹框仅在此配置且用户主动取消 PWA 提示时触发，其他场景均不出现。
条件交互
支持 PWA → 系统提示 → 确认自动安装 PWA 并打开，引导弹窗保留
支持 PWA → 系统提示 → 取消弹出「下载 APP」确认弹框
• 确认 → 关闭弹框，触发 APK 下载，显示进度条；不显示任何 toast 提示
• 放弃 → 关闭弹框，引导安装弹窗保留
不支持 PWA直接触发 APK 下载，显示进度条（0→100%）；不弹出 APK 确认弹框

安卓复制链接弹框（适用于 PWA / PWA+APK 配置且浏览器不支持 PWA 时）

安卓设备在 PWA 或 PWA+APK 安装类型下，若检测到当前浏览器不支持 PWA（无 beforeinstallprompt 事件），则弹出复制链接引导弹框，引导用户在 Chrome 中打开安装。此弹框对下载按钮（顶部横幅 / 底部浮层）和引导安装弹窗均适用。

弹框内容：说明文案（固定，见第五章）+ 链接展示框（左侧 Chrome 图标 + 右侧当前页面 URL，超出截断省略）+ 取消按钮 + 复制链接按钮。链接来源：动态读取当前页面 URL。

点击复制链接后，链接复制到剪贴板，同时显示「Cópia Bem-sucedida」居中 toast 提示（2 秒后自动消失）。toast 关闭不影响底层顶部横幅或底部浮层。

弹窗关闭逻辑

弹框间隔时间配置关闭方式
0小时（强制安装）无法关闭：无 × 按钮，无法点击弹窗外区域关闭。下载/安装完成后弹窗仍保留，直至页面刷新或跳转
1～24小时下载/安装完成后弹窗保留，用户可点击弹窗外区域关闭。距上次关闭超过配置时长后再次弹出
引导安装弹窗与下载按钮（顶部横幅 / 底部安装浮层）交互逻辑完全一致，仅入口形式不同。用户完成 PWA 或 APK 安装后，引导安装弹窗与底部安装浮层不再弹出；顶部横幅保持可见，用户可点击「×」手动关闭（与 3.2 节口径一致）。

3.5 iOS 安装流程

iOS 设备点击安装按钮（顶部横幅 / 底部安装浮层 / 引导安装弹窗）后，根据后台配置的苹果安装类型进入对应流程。

3.5.1 苹果商店选项

点击安装按钮后，跳转至后台配置的 App Store 页面地址（普通地址）或 OneLink 地址。因当前暂不支持，后台「苹果商店」选项已暂时隐藏（不可配置），本节流程作为功能预留说明保留，待选项恢复后生效。

3.5.2 保存桌面 — 非 Safari 浏览器

iOS 设备检测到当前浏览器非 Safari 时，点击安装按钮弹出复制链接引导弹框。

弹框内容：说明文案（固定，见第五章）+ 链接展示框（左侧 Safari 图标 + 右侧链接 URL，超出截断省略）+ 取消按钮 + 复制链接按钮。

链接组成规则：链接来源于系统管理 → 域名管理中，落地页类型为「防封苹果页」的域名 + 固定后缀（后缀因域名类型不同而不同）。若配置了多个防封苹果页域名，每次点击安装时随机选取其一。用户打开链接后进入的仿苹果商店页，样式读取该域名在域名管理中配置的仿苹果商店页模板，详见《仿应用商店模块 PRD》。

从顶部横幅点击安装按钮触发本弹框时，顶部横幅保持可见，不隐藏。
区别说明：本节（下载按钮 / 底部浮层）为「点击安装时」弹出复制链接弹框；而经下载页（仿苹果页）进入时，iOS 非 Safari 为「页面加载时即同步弹出」。二者按各自入口场景区分，详见《仿应用商店模块 PRD》。

3.5.3 保存桌面 — Safari 浏览器

iOS 设备检测到当前浏览器为 Safari 时，点击安装按钮隐藏顶部横幅并整页进入《仿应用商店模块》的仿苹果页（完整页，含数据安全、评分与评价、更新内容、相似游戏、底部等区块）。仿苹果页的页面内容与样式来源详见《仿应用商店模块 PRD》。下方 ②③④ 弹框/浮层与仿应用商店模块为同一套共用组件。

交互流程：

步骤说明
① 仿苹果页展示完整仿苹果页（App 信息、统计、宣传图、数据安全、评分与评价、更新内容、相似游戏、底部等），内容来源：后台仿应用商店配置。点击 CTA 下载按钮触发确认弹框。
