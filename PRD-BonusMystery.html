一、需求背景

神秘彩金是一种面向注册会员的留存激励活动，通过在31天周期内设置5个特定活动日（第2/3/7/15/30天），以充值金额和打码量双维度条件匹配档位，给予会员区间随机奖励，达成提升留存率、促进充值转化的经营目标。

本活动适用于以下场景：

  • 新会员激励：从注册当日起算周期，引导新会员在关键节点完成充值与打码，形成活跃习惯。
  • 老会员激活：以活动开始时间为周期起点，通过阶梯奖励刺激沉默用户回流。
  • 长周期留存：31天跨越用户从尝鲜到依赖的关键周期，有效降低早期流失率。

活动支持两种重置模式：一次性（参与一次后结束）与循环（每32天自动续期），满足不同运营策略需求。

二、名词解释

术语说明
活动周期本活动的计算周期，共31天。第32天触发重置。
周期起点新会员 = 实际注册时间；老会员首次参与 = 活动开始时间；老会员重入 = 再次进入活动时间（详见 5.3）
活动日周期内指定的可领取彩金日，共5个：第2/3/7/15/30天
可领取时间周期起点 + (N-1) 天 00:00:00，即对应活动日当天零点整
档位后台配置的充值门槛分级，每个档位以充值金额下限(≥)区分
领取条件档位内的打码维度子规则：打码倍数(≥) + 对应奖励金额区间。同一档位可设置多条，不同打码倍数对应不同奖励区间
显示奖励档位在前台展示的奖励区间（最小~最大），仅用于吸引展示，非实际发放金额
稽核倍数领取奖励后提款前提：须完成 奖励金额 × 稽核倍数 的打码量。全局统一配置

申领终端仅手动领取时有效，限制允许发起领取操作的终端类型
申领限制除终端外的额外申领门槛，如绑定收款方式、IP唯一性、充值条件等
稽核限定平台稽核打码量仅计算指定游戏平台的有效投注
区间随机奖励系统在可领取时间到达时，按匹配档位及领取条件确定奖励区间后随机生成固定金额，非点击时随机

三、后台功能说明

3.1 活动列表页

路径：后台管理 > 福利中心

本需求涉及以下三个操作入口：

  • 新增活动：点击"新增活动"按钮，弹出活动类型选择弹框，选择"神秘彩金"后进入配置弹框（详见 3.2）。
  • 查看活动：点击列表中的"查看"操作，以只读方式展示活动完整配置，字段与新增/编辑弹框一致，不可修改。
  • 修改活动：点击列表中的"编辑"操作，进入与新增相同的配置弹框，字段可修改。

3.2 新增/编辑活动弹框

弹框为单一滚动表单，字段按以下顺序展示：

字段类型说明
活动分类下拉选择读取系统活动分类配置，下拉选择。编辑已生效活动时不可修改。
活动类型下拉选择固定选择"神秘彩金"，决定活动的业务逻辑和配置项。编辑已生效活动时不可修改。
活动名称单选 + 文本系统默认：自动填入"神秘彩金活动"，前台展示该名称；自定义：可手动输入活动名称
活动时间单选 + 日期永久：活动长期有效，无结束时间；自定义：需填写活动时间范围（控制活动是否有效）及展示时间范围（控制活动在前台的显示时段，可与活动时间不同）
重置方式单选一次性：一个会员只可以参与一次，周期长度为31天，周期结束后不再参与；循环：每31天为一个周期，一个周期结束后自动进入下一个周期
派发方式单选自动派发：系统检测到会员满足奖励条件后，自动将彩金发放至会员账户，无需会员操作（神秘彩金固定使用此方式）；玩家申请：会员需主动发起申请后才能领取（当前不支持，选项置灰）
审核方式单选系统(周期内)：在活动周期内由系统自动完成审核并派发奖励（神秘彩金固定使用此方式）；系统(过期时)：到达奖励过期时间时系统统一审核（当前不支持）；人工审核：由运营人员手动审核后派发（当前不支持）
派奖时间单选立即到账：奖励审核通过后即时发放至会员账户（神秘彩金固定使用此方式）；延迟到账：在指定时间后到账（当前不支持）
领取方式单选自动领取：系统检测到会员满足活动日条件后自动发放彩金，会员无需进入前台操作；手动领取：会员需在活动详情页主动点击"领取"按钮，系统再进行审核派发；选择手动领取时，下方显示申领终端与申领限制配置
奖励作废单选 + 数字过期自动派发：到达过期时间后，不满足条件的奖励自动派发给会员；过期作废：到达过期时间后，不满足条件的奖励自动作废；奖励保留天数自动派发：在设定的保留天数内满足条件后自动派发奖励（选中后需填写保留天数）；奖励保留天数作废：超过保留天数后不满足条件则奖励自动作废（选中后需填写保留天数）
稽核倍数数字输入奖励金额 × 稽核倍数 = 提款所需打码量；全局统一，适用于所有活动日
参与条件单选 + 档位配置全部会员 / 新注册会员（活动时间内注册玩家）；下方为活动日档位配置区，详见下文
申领终端多选仅领取方式为"手动领取"时显示。可选：苹果H5 / 安卓H5 / PWA
申领限制多选 + 条件输入仅领取方式为"手动领取"时显示。各选项说明：
• 完成首充可申领：账号需完成首次充值，才能领取奖励
• 同登录IP只能申领1次：若不同账号登录时的IP相同，某个账号领取奖励后，其他账号在同一活动中无法领取
• 同注册IP只能申领1次：若不同账号注册时的IP相同，某个账号领取奖励后，其他账号在同一活动中无法领取
• 最近N天内充值次数≥N次：领奖当日前N天（含当天）内累计充值N次，才能领取奖励
• 最近N天内充值金额≥N：领奖当日前N天（含当天）累计充值金额满N元，才能领取奖励
注：「绑定收款方式」与「同姓名只能申领1次」因系统暂不支持，始终置灰不可选。
稽核限定平台单选 + 多选不限制 / 仅限以下平台；选择限定时，可按电子/体育分类选择具体游戏平台
规则说明单选 + 文本域系统默认（使用内置规则文案）/ 自定义（可编辑规则内容）
显示排序数字输入控制活动在列表中的展示顺序
活动详情页选择单选跟随皮肤：活动详情页样式跟随商户整体UI皮肤设置；样式一：使用固定的活动详情页模板（即前台原型所展示的页面），选择后可点击预览
背景样式单选 + 选图/上传系统默认（从预设渐变色图库中选择）/ 自定义（上传图片，建议尺寸 660×220px，支持 JPG/PNG/SVG）
LOGO样式单选 + 选图/上传仅背景样式为"系统默认"时显示。系统默认（按分类从图标库选择）/ 自定义（上传图片，建议尺寸 300×300px）
宣传图预览实时预览实时合成展示背景 + LOGO + 活动名称叠加效果

档位配置区（参与条件下方，按活动日分别配置）

顶部固定5个活动日标签（第2日 / 第3日 / 第7日 / 第15日 / 第30日），点击切换对应配置，各活动日相互独立。每个活动日默认展示8个档位，可新增/删除，系统按充值金额从高到低自动排序。

每个档位采用两级结构：

层级字段说明
第一层 — 档位充值金额(≥)该档位充值门槛下限
显示奖励（最小值 ~ 最大值）前台展示的奖励区间，仅用于吸引展示
第二层 — 领取条件（每档可配置多行）打码倍数(≥)满足该条件所需的打码倍数下限
奖励金额（最小值）实际发放奖励区间下限
奖励金额（最大值）实际发放奖励区间上限；系统在此区间内随机生成固定金额
📌 可领取时间到达时，系统先按充值金额从高档向低档匹配档位，再按打码倍数从高到低匹配领取条件，在命中条件的奖励区间内随机生成固定金额。奖励金额在可领取时间点即已确定，非会员点击时随机。

五、前台功能说明

5.1 入口与页面结构

神秘彩金活动入口位于平台优惠列表，会员点击进入活动详情页。

页面整体结构从上至下依次为：

层级内容
顶部导航栏返回按钮 + 页面标题"神秘彩金活动"
宣传图由后台上传配置生成的横幅，合成展示背景图、LOGO 与活动名称
活动日标签区5个活动日标签，横向排列，详见 5.2
内容区个人信息栏 → 可领取时间框 → 档位表 → 活动规则，详见 5.3
底部按钮固定于页面底部，详见 5.6

页面样式：提供两种视觉样式（跟随皮肤 / 样式一），内容与交互完全一致，仅视觉设计不同，由后台"活动详情页选择"字段控制。

5.2 活动日标签区

横向排列5个活动日标签（第2天 / 第3天 / 第7天 / 第15天 / 第30天）。每个标签为全圆角矩形，当前选中标签高亮显示，其下方显示倒三角指示符（在标签框外）。

红点规则：有可领取奖励（即已到达可领取时间且尚未领取）的活动日，标签右上角显示小圆点。仅已登录状态下显示，未登录时不显示红点。红点与选中高亮相互独立，互不影响。

默认选中规则：

  • 若会员已在周期内：默认选中下一个即将到达可领取时间的活动日
  • 若会员首次打开活动（尚未开始周期）：默认选中第一个标签（第2天）

标签视觉样式：仅有两种视觉状态——正常（默认）与高亮（当前选中）。待解锁、可领取、已领取为逻辑状态，不对应独立的视觉样式，逻辑状态仅影响红点是否显示。

逻辑状态触发条件视觉表现
待解锁可领取时间未到达标签正常；无红点
可领取可领取时间已到达，尚未领取标签正常；已登录时右上角显示红点
已领取已成功领取标签正常；红点消失
选中（叠加）当前激活标签标签高亮 + 下方倒三角指示符

5.4 未登录交互

未登录用户进入活动页后，底部展示"立即参与（Participe agora）"按钮，内容区注册时间与可领取时间均按本次页面打开时刻推算展示，档位表与规则正常展示，无领取交互。

点击"立即参与"后：跳转至首页，并同时弹出登录弹窗。

5.5 领取交互（手动领取）

ℹ️ 本节仅适用于后台配置为手动领取模式。自动领取模式下，系统在可领取时间到达时直接发放奖励，前台不显示领取按钮。

领取途径：会员可通过以下两个入口发起领取：

  • 活动详情页：点击底部"领取"按钮
  • 待领取（Pendente）页面：可领取时间到达且系统已确定奖励金额后，平台待领取页面自动生成一条记录，会员可在此查看确定的奖励金额并点击领取。本需求仅说明此为领取路径之一，Pendente 页面为平台已有功能，不在本需求开发范围内。

两个入口触发相同的申领条件验证逻辑。

场景触发条件前台响应
申领条件不满足申领终端不在允许范围，或申领限制条件未满足显示圆角悬浮卡片，提示可申领的终端列表；点击卡片外任意区域关闭
正常领取满足所有申领条件弹出恭喜弹窗，显示"恭喜您获得 [金额] 神秘彩金"；点击确认关闭；按钮变为禁用态；红点消失

申领限制提示卡片：圆角悬浮卡片，浮于活动日标签区正下方，文案居中。示例文案：

请通过 iOS H5、Android H5、PWA、APK、iOS App、iOS Bookmark、APK Release 申领
